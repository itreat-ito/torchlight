プライバシーポリシー

拡張機能: torchlight
施行日: 2026年3月24日

本プライバシーポリシーは、ブラウザ拡張機能「torchlight」（以下「本拡張機能」）が、利用時にどのように情報を取り扱うかを説明するものです。 本拡張機能は、ローカル・ステージング・本番などの環境を視覚的に区別し、関連する利便機能を提供することで、開発および本番運用の作業を支援するために提供されています。

概要

  • お客様の個人情報を販売しません。
  • 本拡張機能内に広告やアクセス解析用のトラッカーを組み込んでいません。
  • お客様が入力した設定はブラウザに保存されます。Chrome の同期機能を利用している場合、一部の設定は Google の Chrome アカウント設定に従い同期されることがあります。
  • 本拡張機能をChrome ウェブストアからインストールした場合、後述の更新確認用エンドポイントには接続しません。

本拡張機能が取り扱う情報

本拡張機能は、機能の提供に必要な範囲でのみデータを処理します。

  • 閲覧中のウェブページの文脈: 環境表示バナーの表示、任意のページタイトル接頭辞の調整、キーボードショートカットの判定のため、現在のページの URL（ホスト名を含む）および必要に応じて文書タイトルを読み取ります。この処理はお使いのブラウザ内でローカルに行われます。
  • アクティブなタブの情報（拡張機能の UI 利用時）: ポップアップを使用する際、設定した別環境で同等の URL を開いたり、指定した形式でタブ情報をコピーしたりするため、アクティブなタブの URL および Chrome が拡張機能に提供する関連メタデータを読み取ります。
  • お客様が設定した項目: 例として、環境ごとのドメインパターンを含むプロジェクト定義、バナーの見た目、言語、キーボードショートカット、任意のページタイトルルール、クリップボード用の書式、拡張機能の有効／無効などです。これらは Chrome の拡張機能ストレージ API を用いて保存されます。
  • ローカル限定の更新関連の設定（Chrome ウェブストア以外からのインストール時）: ストア外でインストールした場合、繰り返しの案内を避けるため、小さなキャッシュおよび「このバージョンをスキップ」などの設定を端末内に保存することがあります。これらはお客様個人を識別するために用いません。

Chrome のストレージと同期

多くの設定は chrome.storage.sync に保存され、同期が有効な場合は Chrome プロファイルに沿って利用できます。 Chrome にサインインし同期をオンにしている場合、同期データの取り扱いは Google のプライバシーポリシー に従います。本拡張機能が Google アカウントの認証情報を取得することはありません。

更新チェックのキャッシュやスキップしたバージョンの状態など、少量のデータは chrome.storage.local に保存される場合があります。これは端末に留まり、本拡張機能による同期の対象ではありません。

ネットワーク通信と第三者

GitHub（公開 API・条件付き）

本拡張機能が Chrome ウェブストア からインストールされたものでないと判定した場合、新しいバージョンの案内のため、GitHub の公開 API （https://api.github.com/repos/itreat-ito/torchlight/releases/latest）に最新リリースのメタデータを要求することがあります。 当該リクエストに、本拡張機能に入力された個人データを含めることはありません。HTTPS 通信に伴い GitHub が IP アドレスなど標準的な技術情報を受け取る場合があることについては、 GitHub のプライバシーに関する声明 をご参照ください。

Chrome ウェブストアからインストールした場合、この GitHub による更新確認は無効です。更新は Chrome により行われます。

本拡張機能からリンクを辿ってリリースページやダウンロードページ等に移動した場合、それらのサイトは各サイトのポリシーが適用されます。

権限とその理由

  • storage — 設定およびローカルの更新確認状態の保存・読み込みのため。
  • tabs — ポップアップで環境切り替えやタブ情報のコピー等を行う際、アクティブなタブの URL を読み取るため。また、設定変更後に開いているタブへ通知するため。
  • コンテンツスクリプト（マニフェスト宣言） — マニフェストで宣言された一致パターンに基づき、閲覧中のウェブページ上で環境バナー等の UI を表示します。Chrome のインストール画面では「すべてのウェブサイト上のデータの読取りと変更」などと表示されることがありますが、これはコンテンツスクリプトの注入範囲に関する表示です。本拡張機能は、任意のサイトへの広範なネットワークアクセスを付与する host_permissions としての <all_urls> は要求していません。
  • ホスト権限（https://api.github.com/*） — Chrome ウェブストア以外からのインストール時に、上記の条件付き GitHub API リクエストを行うため。

また、利用可能な環境では chrome.management.getSelf() を用いて更新元が Chrome ウェブストアかどうかを判定します。management 権限は要求しません。

データの保存期間

設定は、Chrome で拡張機能のデータを消去する、本拡張機能をアンインストールする、オプションで編集するなど、お客様が削除するまで保持されます。 更新メタデータのキャッシュは、一定時間経過後または上書きにより無効化されます。

お子様について

本拡張機能はお子様を対象としたものではなく、故意にお子様から個人情報を収集することはありません。

本ポリシーの変更

本拡張機能の動作や要件の変更に伴い、本プライバシーポリシーを更新することがあります。 更新時には冒頭の施行日を改訂します。

お問い合わせ

本拡張機能に関するプライバシーに関するお問い合わせは、プロジェクトの Issue トラッカーからご連絡ください。 https://github.com/itreat-ito/torchlight/issues

Privacy Policy

Extension: torchlight
Effective date: March 24, 2026

This Privacy Policy describes how the torchlight browser extension (“Extension”) handles information when you use it. The Extension is built to support development and production workflows by visually distinguishing environments (for example, local, staging, and production) and offering related convenience features.

Summary

  • We do not sell your personal information.
  • We do not run advertising or analytics trackers inside the Extension.
  • Configuration you enter is stored in your browser. If you use Chrome’s sync feature, some settings may be synchronized by Google according to your Chrome account settings.
  • When the Extension is installed from the Chrome Web Store, it does not contact our update-check endpoint described below.

Information the Extension processes

The Extension processes data only as needed to provide its features:

  • Web page context (on pages you visit): To show an environment banner, adjust optional page title prefixes, and evaluate keyboard shortcuts, the Extension reads the current page’s URL (including hostname) and, where applicable, the document title. This processing happens locally in your browser.
  • Active tab information (in the Extension UI): When you use the popup, the Extension reads the active tab’s URL (and related metadata Chrome makes available to extensions) so you can open equivalent URLs in another configured environment or copy tab information using your chosen format.
  • Settings and preferences you configure: For example, project definitions (domain patterns per environment), banner appearance, language, keyboard shortcuts, optional page-title rules, copy-to-clipboard format, and whether the Extension is enabled. These values are stored using Chrome’s extension storage APIs.
  • Local-only update preferences (non–Chrome Web Store installs): If you install the Extension outside the Chrome Web Store, the Extension may store a small cache and a “skipped version” preference locally to avoid repeated prompts. These are not used to identify you.

Chrome storage and sync

Most settings are stored with chrome.storage.sync so they can follow your Chrome profile when syncing is enabled. If you are signed in to Chrome and have sync turned on, Google processes synced data under the Google Privacy Policy. The Extension does not receive your Google account credentials.

A small amount of data (update-check cache and skipped-version state) may be stored with chrome.storage.local, which stays on your device and is not synced by this Extension.

Network requests and third parties

GitHub (public API, conditional)

If the Extension detects that it is not installed from the Chrome Web Store, it may request public release metadata from GitHub’s API (https://api.github.com/repos/itreat-ito/torchlight/releases/latest) to inform you about newer versions. That request does not include personal data you typed into the Extension. GitHub may receive standard technical data (such as IP address) as part of any HTTPS request, as described in GitHub’s privacy statement.

When the Extension is installed from the Chrome Web Store, this GitHub update check is disabled; updates are handled by Chrome.

If you follow links from the Extension (for example, to a release or download page), those sites are governed by their own policies.

Permissions (why they exist)

  • storage — Save and load your settings and local update-check state.
  • tabs — Read the active tab’s URL for popup actions (environment switching, copying tab info) and notify open tabs after settings change.
  • Content scripts (declared in the manifest) — Run on pages that match the manifest’s patterns so the environment banner and related UI can appear. Chrome’s install prompt may describe broad site access (for example, read and change data on websites); that wording reflects where content scripts may run. The Extension does not request broad <all_urls> under host_permissions, which would grant wider programmatic network access to arbitrary origins.
  • Host access (https://api.github.com/*) — Allow the conditional GitHub API request described above for non–Chrome Web Store installations.

The Extension also uses chrome.management.getSelf() where available to detect whether updates come from the Chrome Web Store, without requesting the management permission.

Data retention

Your settings remain until you remove them (for example, by clearing extension data in Chrome, uninstalling the Extension, or editing options). Cached update metadata expires after a limited time or when replaced.

Children

The Extension is not directed at children, and we do not knowingly collect personal information from children.

Changes to this policy

We may update this Privacy Policy when the Extension’s behavior or requirements change. The effective date at the top will be revised when updates are published.

Contact

For privacy questions about this Extension, please contact us through the project’s issue tracker: https://github.com/itreat-ito/torchlight/issues
